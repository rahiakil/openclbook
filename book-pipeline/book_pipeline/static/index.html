Local & published library

Local disk — this table scans BOOK_PIPELINE_PROJECTS_DIR (see —). Nothing here is pulled from S3 automatically: uploads go out via scripts/upload_completed_manuscripts_s3.py (operator/CI); the marketing site reads presigned URLs from your HTTP API. Batch order comes from gutenberg_library/manifest.json (or BOOK_GUTENDEX_MANIFEST) when present — same sequence as gutendex_process_batch.py. There is no “submit book” button in this supervisor UI yet: runs are started from here (full/guided pipeline) or from the CLI; human gates use markdown files (e.g. outputs/plan_for_review.md) plus --prep-resume / plan gate flows where implemented.
